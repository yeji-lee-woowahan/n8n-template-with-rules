# 코드 정리

## 1. 기존 코드베이스 파악

사용자에게 정리 대상을 듣고 관련 코드와 기획서를 파악한다.

- 정리하려는 코드의 현재 구조와 동작 방식 파악
- 관련 기획서(`n8n-template/docs/기획서/`)가 있다면 함께 파악
- 코드와 기획서 간의 불일치 확인

## 2. 검토 내용 검수

파악한 내용을 사용자에게 공유하고 검수받는다.

- 정리 방향과 범위에 대해 합의

## 3. 구현계획 정리

### 간단한 경우

직접 작업을 제안한다.

- 코드 수정 후 `.agent/개발진행/구현계획.md`에 사후 기록
- 완료 후 **5.가상검증** 안내

### 복잡한 경우

`.agent/개발진행/구현계획.md`에 맥락 전달용 정보를 정리한다.

> "구현계획 정리 완료. **2.구현계획** 커맨드를 실행해주세요."
