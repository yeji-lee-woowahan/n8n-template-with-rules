# 구현계획

## 0. 참고 사례 탐색 (새 페이지 생성 시)

새 페이지나 주요 UI를 처음 만드는 경우, 계획 수립 전에 참고 사례를 찾는다.

1. **우리 프로젝트 먼저**: `n8n-template/src/` 에서 비슷한 페이지/컴포넌트가 있는지 탐색
2. **없으면 Atelier 참고용 프로젝트**: `우리프로젝트아님아뜰리에참고용/commerce-backoffice-platform-main/` 에서 비슷한 사례 탐색

찾은 사례의 구조(폴더, 컴포넌트 분리, 훅 패턴 등)를 구현계획에 반영한다.

## 1. 구현 계획 수립

변경된 기획서와 `.agent/개발진행/구현계획.md`를 기반으로 구현 계획을 세운다.

- `git diff`로 변경된 기획서와 구현계획.md 확인
- 기획서의 "필수 확인 동작" 섹션 꼼꼼히 읽기
- 구현에 필요한 핵심 로직과 제약사항 모두 파악
- 다른 세션에서 실행하는 AI가 오해 없이 작업할 수 있는 '구현용 프롬프트' 작성
- 변수명, 파일 경로, 주의사항 포함하여 구체적으로 작성

## 2. 구현계획 승인

구현계획을 사용자에게 공유하고 승인받는다.

- 복잡한 설계가 있으면 설계 원칙을 근거로 제시
- 사용자와 핑퐁하여 최종 계획 확정

## 3. 구현계획.md 업데이트

최종 구현계획을 `.agent/개발진행/구현계획.md` 아래쪽에 추가한다.

- 프롬프트만으로 계획대로 누락 없이 작업 가능하도록 작성
- 참고 문서 링크, 계획 설명, 맥락, 주의사항 포함
- 설계를 망치지 않는 선에서 AI가 자유롭게 구현할 수 있도록 허용
- 토큰 사용량 고려 — 필요시 여러 프롬프트로 분할
- 여러 세션 시 맨 위에 공통 맥락, 아래에 번호별 프롬프트

> "구현계획 작성 완료. **3.설계검토** 커맨드를 실행하여 설계를 검토해주세요."
