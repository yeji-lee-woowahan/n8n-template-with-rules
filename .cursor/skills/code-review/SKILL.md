# Code Review 스킬

> 코드 변경 후 체크리스트 기반으로 코드 품질을 검증한다.

## 사용 시점

- 구현 완료 후 커밋 전
- 커맨드 `6.코드리뷰`에서 호출

## 프로세스

### 1. 변경된 파일 수집

```bash
cd n8n-template && git diff --name-only
```

- 변경된 파일이 없으면 "검증할 변경사항이 없습니다" 안내 후 **중단**

### 2. 변경 유형 분류

| 유형 | 경로 패턴 |
|------|-----------|
| 기획서 | `docs/기획서/**/*.md` |
| 프론트 코드 | `src/**/*` |

### 3. 체크리스트 검증

변경 유형에 따라 체크리스트를 실행한다.

- **기획서 변경 시**: `n8n-template/docs/기획서/0-기획서-작성가이드.md`를 읽고 양식 준수 여부 확인
- **프론트 코드 변경 시**: [frontend-checklist.md](./frontend-checklist.md)를 읽고 항목별 검증

변경된 코드의 diff를 실제로 읽으며 체크리스트 각 항목을 검증한다. 체크박스를 기계적으로 채우지 말고, 실제 코드에서 위반 사항이 있는지 확인한다.

### 4. 결과 보고

#### 모두 통과

> "코드리뷰 완료. 모두 통과했습니다.
> **7.비판적검토** 커맨드를 실행해주세요."

#### 수정 필요

1. 미통과 항목과 수정 방향을 정리한다
2. `.agent/개발진행/구현계획.md`를 업데이트한다:
   - 기존 세션은 건드리지 않음
   - 맨 아래에 **수정 세션**을 추가하여 수정 내용 명시
3. 사용자에게 수정 진행을 요청한다

> "수정이 필요합니다. 구현계획.md에 수정 세션을 추가했습니다.
> **4.구현** 커맨드로 수정 후 다시 **6.코드리뷰**를 실행해주세요."
