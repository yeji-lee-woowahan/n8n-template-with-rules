# 스타일링

## 핵심 규칙: 몰드어드민에서 import

**★★★★★ `@atelier/blue` 직접 사용 금지**

몰드어드민(`@atelier-mold/admin`)이 블루를 re-export하므로 **모든 컴포넌트는 `@atelier-mold/admin`에서 import**합니다.

```tsx
// ❌ Bad - @atelier/blue 직접 import
import { Flex, Typography, Button } from '@atelier/blue'

// ✅ Good - @atelier-mold/admin에서 import
import { Flex, Typography, Button } from '@atelier-mold/admin'
```

**이유:**
- 몰드어드민의 `AtelierRoot`가 필요한 Provider를 포함
- `@atelier/blue`와 `@atelier-mold/admin`을 혼용하면 Provider 불일치 에러 발생
- `style.css` import는 `AtelierProvider.tsx`에서 한 번만 수행

---

## 원칙

- **Atelier 우선** - Atelier 컴포넌트 props로 해결 가능하면 props 사용
- **스타일 오버라이드 금지** - Atelier 컴포넌트는 props로만 스타일 변경
- **Tailwind 금지**
- **.css 파일 직접 작성 금지** - SCSS Module 사용
  - 예외: `src/app/styles/` (전역 스타일 전용 - reset.css, fonts.css 등)
- **인라인 스타일 최소화** - Atelier props → SCSS → 간단한 건 인라인 OK

---

## Atelier 스타일 변경 순서

### 1. Props 확인 (권장)

Atelier 컴포넌트는 **props로만 스타일 변경**합니다. 커스텀이 필요하면:

1. **MCP로 최신 props 검색** - 적절한 prop이 있으면 사용
2. **div로 감싸서 해결** - props로 안 되면 wrapper div에 스타일 적용
3. **스펙아웃 요청** - 그래도 안 되면 기획자에게 스펙아웃 요청

```tsx
// ✅ Good - props 사용
<Button variant="fill" color="primary" size="large" width={200}>
  확인
</Button>

// ✅ Good - wrapper div로 해결
<div className={styles.buttonWrapper}>
  <Button variant="fill">확인</Button>
</div>

// ❌ Bad - 스타일 오버라이드 시도
<Button style={{ backgroundColor: 'red' }}>확인</Button>
```

### 2. Flex로 레이아웃

정렬이 필요하면 Atelier `Flex` 컴포넌트 사용:

```tsx
// ❌ Bad - div + CSS flexbox
<div style={{ display: 'flex', alignItems: 'center' }}>

// ✅ Good - Atelier Flex 컴포넌트
<Flex alignItems="center" justifyContent="space-between" gap="100">
  <Typography>제목</Typography>
  <Button>확인</Button>
</Flex>
```

### 3. Typography로 텍스트

텍스트는 HTML 태그 대신 `Typography` 사용:

```tsx
// ❌ Bad
<span>텍스트</span>
<p style={{ color: 'gray' }}>설명</p>

// ✅ Good
<Typography as="span" variant="body1">텍스트</Typography>
<Typography as="p" variant="body2" color="fg_secondary">설명</Typography>
```

**금지:**
- `!important`
- Atelier 내부 클래스 직접 오버라이드
- SCSS에서 Atelier 컴포넌트 스타일 강제 변경

## CSS Module

### 파일 규칙

| 항목 | 규칙 |
|------|------|
| 파일명 | `ComponentName.module.scss` |
| 위치 | 컴포넌트와 같은 디렉토리 |
| 클래스명 | camelCase (`headerTitle`, `submitButton`) |

### 스타일 방식 선택

| 상황 | 방식 |
|------|------|
| 정적 스타일 | SCSS Module |
| 동적 스타일 (JS 값) | inline style |
| JS → CSS 값 전달 | CSS 변수 |

```tsx
// 동적 값은 CSS 변수로 전달
<div
  className={styles.progress}
  style={{ '--progress-width': `${percent}%` } as React.CSSProperties}
/>
```

```scss
.progress {
  width: var(--progress-width);
}
```

### 조건부 클래스 (clsx)

```tsx
import clsx from 'clsx';

className={clsx(styles.btn, { [styles.active]: isActive })}
className={clsx(styles.btn, isActive && styles.active)}
```

### 인라인 스타일 vs SCSS 분리 기준

| 조건 | 방식 |
|------|------|
| 스타일 속성 1~3개, 단순 | 인라인 style prop |
| 스타일 속성 4개 이상 | SCSS 분리 고려 |
| 한 파일에 인라인 스타일 3군데 이상 | SCSS 분리 |
| hover, 미디어쿼리 필요 | SCSS 필수 (인라인 불가) |

**예시:**

```tsx
// ✅ 3개 이하 - 인라인 OK
<Flex style={{ minWidth: 1280, maxWidth: 1280, margin: '0 auto' }}>

// ⚠️ 4개 이상 - SCSS 분리 고려
// → ComponentName.module.scss로 분리
```

**복잡도 우선:** 개수보다 복잡도가 더 중요할 때도 있음
- 단순 값 4~5개 → 인라인도 OK
- 복잡한 값 3개 → 분리가 나음

### 하드코딩 금지

핵심 색상(Primary/Brand)은 Atelier 토큰 또는 CSS 변수 활용. 단순 UI 수치(px)는 리터럴 허용.
