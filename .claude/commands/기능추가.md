## 1. 기획 파악 및 싱크

사용자에게 구현하고자 하는 내용을 듣고 기획의 구멍을 찾는다.

- 기존 기획서(`n8n-template/docs/기획서/`)를 읽고 배경 파악
- 새 스펙을 머릿속으로 그린 뒤 확정에 필요한 정보를 묻고 싱크
- UX/사용성/기술적 측면에서 피드백

## 2. 기획서 작성

스펙 확정 후 기획서를 수정한다.

- `n8n-template/docs/기획서/0-기획서-작성가이드.md`를 참고하여 양식에 맞게 작성
- 작성 내용을 사용자에게 공유하고 컨펌 후 파일 수정

## 3. 기획서 검토

작성된 기획서를 검토한다.

- `n8n-template/docs/기획서/0-기획서-작성가이드.md` 양식 준수 여부 확인
- 기존 기획서와 대조하여 빠진 플로우, 예외처리, 엣지 케이스 확인
- 사용자에게 공유하고 수정

## 4. 구현계획 정리

### 간단한 경우

구현계획 작성 없이 직접 작업을 제안한다.

- 코드 수정 후 `.agent/개발진행/구현계획.md`에 사후 기록
- 완료 후 바로 **/가상검증** 안내

### 복잡한 경우

`.agent/개발진행/구현계획.md`에 맥락 전달용 정보를 정리한다.

- 고려했던 것들과 맥락, 주의사항 포함
- 참고 파일은 경로만 기재 (디테일 X)

> "구현계획 정리 완료. **/구현계획** 커맨드를 실행해주세요."
