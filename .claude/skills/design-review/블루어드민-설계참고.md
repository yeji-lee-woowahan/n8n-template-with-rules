# 블루어드민 설계 참고

## SearchFilter 패턴

### SearchFilter를 react-hook-form과 함께 사용할 때는 FormProvider로 감싸기

SearchFilter 컴포넌트는 내부에 `<form>` 태그를 이미 렌더링한다. HTML에서 form 안에 form을 중첩할 수 없으므로, react-hook-form을 쓸 때 `<form>`을 또 만들면 안 된다.

FormProvider는 form 태그 없이 컨텍스트만 제공하므로 충돌 없이 사용 가능하다.

```tsx
const methods = useForm<SearchFormData>()

<FormProvider {...methods}>
  <SearchFilter onSubmit={methods.handleSubmit(handleSubmit)}>
    <SearchFilter.Contents>
      <SearchFilter.Field>
        <SearchFilter.Label>검색어</SearchFilter.Label>
        <SearchFilter.Control>
          <FormInput name="keyword" control={methods.control} />
        </SearchFilter.Control>
      </SearchFilter.Field>
    </SearchFilter.Contents>
    <SearchFilter.Footer>
      <Button type="reset" onClick={() => methods.reset()}>초기화</Button>
      <Button type="submit">검색</Button>
    </SearchFilter.Footer>
  </SearchFilter>
</FormProvider>
```

### 검색 조건은 URL Query Params로 관리하기

검색 조건을 컴포넌트 state가 아닌 URL에 저장하면:
- 뒤로가기 시 이전 검색 결과 복원
- 새로고침해도 상태 유지
- URL 공유로 동일 검색 결과 공유 가능

useForm + useQueryParam 조합을 매번 작성하지 않고 공통 훅으로 추상화하면 편리하다.

```tsx
// 공통 훅
const { methods, onSubmit, onReset } = useCommonSearchFilter({
  defaultValues,
  serialize,    // Form → QueryParams 변환
  deserialize,  // QueryParams → Form 변환
})

// submit 시 URL 업데이트 → 테이블이 URL 변경 감지해서 자동 refetch
```

### SearchFilter.Label 너비 조정

기본값은 레이블이 전체 너비를 차지한다. `width` prop으로 레이블 너비를 조정할 수 있다.

| 용도 | width 값 | 예시 |
|------|----------|------|
| 짧은 레이블 | `80` | 키워드, 정렬 |
| 일반 서브 레이블 | `100` | 셀러 ID, 브랜드명 |
| 긴 서브 레이블 | `110~152` | 사업자등록번호, 카탈로그 아이템ID |
| 레이블 숨기기 | `0` | 중첩 필드에서 레이블 없이 컨트롤만 표시 |

```tsx
// 기본 레이블
<SearchFilter.Label width={80}>키워드</SearchFilter.Label>

// 중첩 필드에서 서브 레이블
<SearchFilter.Label width={100}>셀러 ID</SearchFilter.Label>

// 레이블 숨기기
<SearchFilter.Label width={0} />
```

컨트롤(Input, Select) 너비도 함께 지정하면 레이아웃이 컴팩트해진다.

```tsx
<TextInput width={328} />  // 일반적인 입력 필드
<Select width={180} />     // 셀렉트 박스
```

### SearchFilter가 비슷하게 생긴 경우 Row 단위로 공통화하기

전체 SearchFilter를 공통 컴포넌트로 만들면 props가 복잡해지고 유연성이 떨어진다. 페이지마다 필드 조합이 다르기 때문이다.

대신 Row(검색 필드 한 줄)를 렌더 함수로 분리하고, 페이지에서 필요한 Row만 조합한다.

```tsx
// Row 렌더 함수 정의
export const renderKeywordRow = (methods: UseFormReturn<SearchForm>) => (
  <SearchFilter.Field>
    <SearchFilter.Label>검색어</SearchFilter.Label>
    <SearchFilter.Control>
      <FormInput name="keyword" control={methods.control} />
    </SearchFilter.Control>
  </SearchFilter.Field>
)

// 페이지에서 필요한 Row만 조합
<SearchFilterWrapper onSubmit={onSubmit} onReset={onReset}>
  {renderKeywordRow(methods)}
  {showCategory && renderCategoryRow(methods)}
  {renderDateRow(methods)}
</SearchFilterWrapper>
```

SearchFilter 레이아웃(Contents + Footer)이 동일하다면 래퍼 컴포넌트로 분리할 수도 있다.

```tsx
// SearchFilterWrapper - 레이아웃 + 버튼 공통화
export const SearchFilterWrapper = ({ children, onSubmit, onReset }) => (
  <SearchFilter onSubmit={onSubmit}>
    <SearchFilter.Contents>{children}</SearchFilter.Contents>
    <SearchFilter.Footer>
      <Button onClick={onReset}>초기화</Button>
      <Button type="submit">검색</Button>
    </SearchFilter.Footer>
  </SearchFilter>
)
```
