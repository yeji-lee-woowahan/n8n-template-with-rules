# 주석 작성 가이드

## 대원칙

> **코드는 What(무엇을), 주석은 Why(왜)를 설명한다.**

코드를 그대로 한글로 번역하는 '중계형 주석'은 절대 금지.

```typescript
// BAD - 중계형 주석
const MAX_RETRY = 3; // 최대 재시도 횟수를 3으로 설정

// GOOD - Why를 설명
// [SPEC] 네트워크 불안정 환경 고려하여 3회 재시도 (기획서: n8n-template/docs/기획서/XX.md)
const MAX_RETRY = 3;
```

## 태그 규칙

### `[SPEC]` - 기획/비즈니스 요구사항

코드만 봐서는 이해되지 않는 수치나 로직에 사용.

**필수:** 기획서 링크 또는 근거 명시

```typescript
// [SPEC] 무료 사용자는 MCP 3개까지만 등록 가능 (기획서: n8n-template/docs/기획서/엔티티.md)
const MAX_MCP_FREE = 3;

// [SPEC] 메시지 삭제는 작성 후 5분 이내만 가능 (기획: PM 요청 2024-01)
if (now - createdAt > 5 * 60 * 1000) {
  throw new Error('삭제 가능 시간 초과');
}
```

**[SPEC]이 필요한 상황:**
- 뜬금없는 매직 넘버(숫자)나 예외 처리
- 직관과 반대되는 로직 (개발 상식으로는 A지만 기획상 B로 구현)
- 백엔드 API 미지원/레거시 데이터 문제로 인한 우회 처리
- 수정 시 다른 기능에 Side Effect가 발생하는 코드

---

### `[CORE]` - 기술적 의사결정

성능 최적화, 라이브러리 버그 우회, 비표준 구현 등에 사용.
설계문서를 대체하여 코드에 직접 설계 의도를 남기는 역할도 수행.

**필수:** 왜 굳이 이렇게 짰는지에 대한 이유

```typescript
// [CORE] useLayoutEffect 사용 - 스크롤 위치 복원이 화면 깜빡임 없이 동기적으로 처리되어야 함
useLayoutEffect(() => {
  scrollRef.current?.scrollTo(savedPosition);
}, []);

// [CORE] JSON.parse 대신 직접 파싱 - 10만 건 이상에서 30% 성능 개선 (벤치마크: PR #123)
function parseCustomFormat(data: string) { ... }

// [CORE] Atelier Dialog 포커스 트랩 버그 우회 (이슈 링크)
```

#### [CORE] 3레벨 위치 기준

| 레벨 | 위치 | 설명 | 예시 |
|------|------|------|------|
| 1 | 모듈/훅 정의부 | 핵심 책임, 의존 관계, 호출자 | 커스텀 훅, 유틸 모듈, feature 훅 |
| 2 | 다단계 흐름 함수 | API→상태→UI 같은 cross-cutting 흐름 요약 | 폼 제출, 검색 필터 처리 등 |
| 3 | 설계 결정 분기점 | 코드만 봐서는 '왜'를 알기 어려운 곳 | 특정 조건 분기, 우회 처리 등 |

#### JSDoc 형태 [CORE] 예시

```typescript
/**
 * [CORE] MCP 등록/수정 폼 상태 관리
 *
 * 의존: react-hook-form, useMcpDetailQuery
 * 호출자: McpCreatePage, McpEditPage
 *
 * 설계 결정:
 * - 등록/수정 공유 훅 (isEditMode로 분기)
 * - API 응답 ↔ 폼 값 양방향 변환 (transformToFormValues / transformToRequest)
 * - 폼 상태가 유일한 상태 소스 (별도 useState로 이중 관리하지 않음)
 */
export function useMcpForm({ currentData }: UseMcpFormProps) { ... }

/**
 * [CORE] 검색 필터 처리 흐름: 폼 입력 → 조회 확정 → API 호출
 *
 * 1. 사용자가 폼 필드 편집 (로컬 state, 실시간 반영)
 * 2. "조회" 버튼 클릭 → submittedValues 확정
 * 3. submittedValues + URL params(sort, page) → API 쿼리 파라미터로 결합
 */
export function useSearchFilter<Form>({ defaultValues }: Props) { ... }
```

#### 진입점 컨펌 플로우

문서정리 스킬에서 [CORE] 주석을 작성할 때:
1. 변경 코드에서 3레벨 기준으로 [CORE] 주석 후보 위치를 나열
2. 사용자에게 "이 위치들에 [CORE] 주석을 추가할까요?" 확인
3. 승인된 위치에만 주석 작성
4. 과도한 주석 방지 — 코드로 충분히 이해 가능한 곳에는 달지 않음

## 주석을 달기 전에

1. **변수/함수명을 더 명확하게 바꿀 수 있는가?**
   - 좋은 이름이 주석보다 낫다

2. **코드를 읽으면 알 수 있는 내용인가?**
   - 그렇다면 주석 불필요

3. **코드를 수정했는가?**
   - 관련 주석도 함께 수정 또는 삭제
