# 문서 작성 규칙

## 기획서

`n8n-template/docs/기획서/0-기획서-작성가이드.md`를 반드시 참고하여 양식에 맞게 작성한다.

### 핵심 원칙

- 한 섹션 = 한 구현 단위 (화면에서 손가락으로 가리킬 수 있는 UI 블록)
- 각 섹션은 **컴포넌트** → **동작** → **API** 순서로 작성
- API는 문서 끝에 몰지 않음 — 각 섹션의 동작 바로 아래에 배치
- 모달/패널/탭은 항상 별도 섹션으로 분리
- 섹션 크기: 약 40~50줄 이내 권장, 넘으면 분리 검토
- 파일 크기: 500줄 초과 시 파일 분리 검토

### 작성 전 확인

- `n8n-template/docs/기획서/2-공용레이아웃.md` — 적용할 레이아웃 확인
- `n8n-template/docs/기획서/2-공용컴포넌트.md` — 사용할 공용 컴포넌트 식별

## 주석
