# 구현계획

> 작업 시작 전 계획을 여기에 작성하고, 완료 후에는 양식만 남기고 내용을 비운다.

---

## 작업 순서 개요

```
Phase 2 (서버 API 명세서)
  ↓ 완료 후
Phase 1-A (순수 프론트) ←→ 서버 개발 (동시 진행)
  ↓ 서버 완료 후
Phase 1-B (서버 연동 필요 작업) + Phase 3 (API 연동)
```

mock으로 처리된 API + 신규 필요 API를 정리한다.

### 산출물

`docs/` 하위에 API 명세 문서 (양식: `METHOD /path` + body param + result)
