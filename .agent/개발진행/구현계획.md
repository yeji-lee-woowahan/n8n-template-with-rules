**기술 스택**: React 18, Vite 7, TypeScript 5.9, React Router 7, TanStack React Query 5, Zustand, react-hook-form, @atelier-mold/admin (UI), Axios, Sass
**FSD 구조**: `app/` → `pages/` → `features/` → `shared/` (entities 레이어 없음, DTO는 `shared/api/model`에서 관리)
**참고 프로젝트**: `/Users/yeji.lee/dev/MCPHUB-with-rules/MCPHUB/src` — 동일 FSD 패턴, 동일 아뜰리에 UI

**현재 상태**:
- app shell 완성 (layouts, providers, routes, styles)
- pages 4개 스텁 (TemplateGalleryPage, GeneratePage, TemplateCreatePage, TemplateDetailPage)
- `features/` 비어있음, `shared/ui/` 비어있음
- orval 설정 완료 (`orval.config.ts`), 생성 코드 아직 없음

**API 문서**: `https://n8n-template-api.betawoowa.in/swagger-ui/index.html`
**기획서 위치**: `n8n-template/docs/기획서/`
**MCPHUB 참고 위치**: `/Users/yeji.lee/dev/MCPHUB-with-rules/MCPHUB/src`

**orval 생성 패턴** (MCPHUB과 동일):
- `shared/api/generated/{tag}/{tag}.ts` — 태그별 API 훅 (useQuery, useSuspenseQuery)
- `shared/api/model/` — OpenAPI 스키마 기반 타입
- `customInstance` 뮤테이터 사용, `useMutation: false` (mutation은 수동 구현)
- feature hooks에서 orval 훅을 래핑하여 편의 접근자 제공

**FSD 규칙**:
- `pages/` = 라우트 + 조합만 (로직 없음)
- `features/` = 도메인 UI + hooks + 타입/상수
- `shared/` = 교차 관심사 (API, 공용 UI, 공용 hooks)
- barrel export (`index.ts`) 사용, import는 `@features/template`, `@shared/ui` 형태

**라우트 경로 정리** (기획서 기준으로 통일):

| 경로 | 페이지 | 비고 |
|------|--------|------|
| `/` | HomePage | 기존 TemplateGalleryPage → `git mv`로 리네임 |

| `/templates/create` | TemplateCreatePage | 기존 `/templates/new` → 경로 변경 |
| `/templates/edit/:id` | TemplateEditPage | 신규 |

**주의사항**:
- 파일 이동 시 반드시 `git mv` 사용
- 아뜰리에 컴포넌트 사용법은 MCPHUB 코드와 `@atelier-mold/admin` 패키지 참고
- MSW 핸들러로 로컬 개발 모킹 (타입은 orval이 생성한 것 사용)
- 구현 완료 후 `pnpm typecheck`으로 타입 확인

### 세션 0: orval API 코드 생성

**목표**: Swagger 문서에서 orval로 API 훅과 타입을 자동 생성한다. 이후 모든 세션에서 이 타입/훅을 기반으로 작업한다.

**작업 내용**:

1. **orval 실행**
   - `API_DOCS_BASE=https://n8n-template-api.betawoowa.in pnpm generate:api`
   - 생성 위치: `shared/api/generated/` (태그별 훅), `shared/api/model/` (스키마 타입)

2. **생성 결과 확인**
   - 생성된 태그별 파일 목록 확인 (templates, reviews, workflows 등)
   - 주요 타입 확인: 요청/응답 DTO, 엔티티 타입
   - 주요 훅 확인: `useTemplateController*`, `useReviewController*`, `useWorkflowController*`
   - `pnpm typecheck` 통과 확인

3. **orval 설정 고정** — `.env.development`에 `API_DOCS_BASE` 추가 (또는 `orval.config.ts`의 기본값 업데이트)
   - 이후 `pnpm generate:api`만으로 재생성 가능하도록

4. **생성된 API 인벤토리 정리** — 기획서와 대조하여 필요한 API 목록 확인
   - 템플릿 CRUD: 목록/상세/등록/수정
   - 리뷰: 목록/요약/등록
   - 워크플로우: 생성

**완료 기준**: `pnpm typecheck` 통과, `shared/api/generated/`와 `shared/api/model/` 생성 완료
