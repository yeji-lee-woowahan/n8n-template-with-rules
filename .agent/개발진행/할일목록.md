# 구현계획

> 작업 시작 전 계획을 여기에 작성하고, 완료 후에는 양식만 남기고 내용을 비운다.

---

Phase 1. [프론트 변경사항]

(홈 페이지)

- 페이지 삭제하고 템플릿 목록이 홈 페이지 역할을 하게 하기
- 홈이 사라지는거니까 영향받는 부분 전부 찾아서 할일 목록 챙겨보기

(템플릿 목록)

- 한번에 나오는 템플릿의 갯수를 100개로 해주기. 페이징 단위를 바꾸는거임
- 정렬방식의 경우 인기순을 맨 처음으로 해주고 인기순은 '조회수' 기준으로 조회수가 많은 템플릿을 맨위로 올라오게 해줄건데 이건 서버 스펙이 될것임

(AI 워크플로우 생성)

- API 가 붙어있지 않은 부분이 혹시 있는지 확인해보기

(커스텀 노드 목록)

- 카테고리 검색은 삭제하기
- 노드 타입은 목록에서 보여주지 말기
- 검색 쪽에서 상태 > 활성 상태로 하고 전체/활성/비활성을 Chip 으로 선택 가능하게 하기. MCPHUB 쪽을 참고
- 검색쪽에 임베딩 상태로도 검색 가능하게 하기. 똑같이 CHIP 으로 추가
- 목록에서는 재시도 버튼 삭제하기
- 목록에서는 관리 버튼들 삭제하기

(커스텀 노드 등록하기)

- 카테고리 입력은 삭제하기
- 스펙 내용에 기본 템플릿으로 아래 내용 initial 값으로 입력해주기

### 설명

이 노드가 하는 일에 대한 설명

### 파라미터

- paramName (타입, 필수여부): 설명

### 입력/출력

- 입력: 이전 노드에서 받는 데이터 형태
- 출력: 다음 노드로 전달하는 데이터 형태

### 사용 예시

이 노드를 사용하는 일반적인 시나리오

### 연결 규칙

- 주로 연결되는 앞/뒤 노드 타입

### JSON 예시

```json
{
  "paramName": "value"
}
```

(커스텀 노드 상세)

- 카테고리 삭제
- 기본정보와 설명 쪽에 들어간 글씨들 폰트 키워주기
- 상태 chip 에 색깔 넣어주기 활성은 초록 성공은 파랑
- 활성 상태 옆 임베딩 상태 옆에 버튼 넣어서 활성/비활성화 하기랑 임베딩 재요청하기 버튼 넣어주기
- 각 버튼 동작한 다음에는 invalidate 를 해줘서 상세에서도 목록에서도 바로바로 반영되어 보이게 해주기

Phase 2. [서버 API 명세서 작성]

- 다른 곳에 있는 서버 프로젝트도 내가 작업하기로 했음
- 프론트에서 orval 로 처리해봤더니 서버 스펙이 안맞는 부분은 mockApi 로 처리했는데 그렇게 mock 으로 되어있는 애들을 정리해서 서버 API 문서 작성해주기.
- docs 하위에 아래 양식으로 뽑아주기.

GET /api/admin/node-specs/{id}
(body param)

(result)

Phase 3. [서버-프론트 API 연동]

- 서버 구축 끝내고 돌아온 상태
- orval 로 서버 작업분 땡기기
- 프론트-서버 연동 하기
