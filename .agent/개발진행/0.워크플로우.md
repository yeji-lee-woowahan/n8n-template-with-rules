# 워크플로우

## 흐름

```
1단계 (진입)           2단계 (설계)          3단계 (구현)         4단계 (검증)
──────────────        ──────────          ──────────         ─────────────────────
1.기능추가    ──→                                              5.가상검증
1.버그수정    ──→      2.구현계획  ──→       4.구현      ──→    → 6.코드리뷰
1.코드정리    ──→        ↓                                     → 7.비판적검토
                     3.설계검토                                 → 8.문서정리
                                                              → 9.커밋
```

## 커맨드 설명

### 1단계: 진입

| 커맨드 | 역할 |
|--------|------|
| 1.기능추가 | 기획 파악 → 기획서 작성 → 검토 → 구현계획 정리 |
| 1.버그수정 | 체계적 디버깅 (systematic-debugging 스킬) |
| 1.코드정리 | 코드 파악 → 검수 → 구현계획 정리 |

### 2단계: 설계

| 커맨드 | 역할 |
|--------|------|
| 2.구현계획 | 계획 수립 → 승인 → 구현계획.md 업데이트 |
| 3.설계검토 | 구현계획의 설계 검증 (design-review 스킬) |

### 3단계: 구현

| 커맨드 | 역할 |
|--------|------|
| 4.구현 | 구현계획.md 기반 구현 |

### 4단계: 검증

| 커맨드 | 역할 |
|--------|------|
| 5.가상검증 | 기획서 기반 AI 동작 검증 (virtual-verify 스킬) |
| 6.코드리뷰 | 체크리스트 기반 코드 검증 (code-review 스킬) |
| 7.비판적검토 | 상태추적 / 깨지기 쉬운 시나리오 검증 (critical-review 스킬) |
| 8.문서정리 | 주석 보강 / 주석 점검 / 기획서 동기화 (doc-cleanup 스킬) |
| 9.커밋 | pnpm verify → Jira 연동 커밋 (commit 스킬) |

## 프로젝트 구조

- 코드: `n8n-template/src/**`
- 기획서: `n8n-template/docs/기획서/`
- 구현계획: `.agent/개발진행/구현계획.md`

## 규칙 체계

| 구분 | 위치 | 적용 방식 |
|------|------|-----------|
| 룰 | `.cursor/rules/*.mdc` | glob 기반 자동 적용 (코드/문서 편집 시) |
| 스킬 | `.cursor/skills/*/SKILL.md` | 커맨드가 호출하는 상세 프로세스 |
| 커맨드 | `.cursor/commands/*.md` | 사용자가 팔레트에서 실행하는 절차 |

### 룰 목록

| 파일 | glob | 역할 |
|------|------|------|
| `frontend.mdc` | `n8n-template/src/**` | 프론트엔드 코드 규칙 (API, 스타일, 상태관리, FSD 등) |
| `docs.mdc` | `n8n-template/docs/**/*.md` | 문서 작성 규칙 (기획서 양식, 주석 가이드) |
| `comment-preserve.mdc` | `n8n-template/src/**` | 코드 수정 시 주석 보존 규칙 |
| `git.mdc` | `n8n-template/**` | Git 환경 (별도 저장소 규칙) |
| `wiki-url.mdc` | always | 위키 URL 접근 규칙 |

### 스킬 목록

| 스킬 | 호출 커맨드 | 역할 |
|------|------------|------|
| code-review | 6.코드리뷰 | 프론트 코드 체크리스트 검증 + 참조문서 7개 |
| design-review | 3.설계검토 | 설계 검토 체크리스트 + 참조문서 5개 |
| commit | 9.커밋 | pnpm verify → Jira MCP → 커밋 |
| critical-review | 7.비판적검토 | 상태추적, 깨지기 쉬운 시나리오, 확장성 |
| doc-cleanup | 8.문서정리 | 주석 보강, 주석 점검, 기획서 동기화 |
| virtual-verify | 5.가상검증 | 기획서 기반 AI 동작 검증 + 자동 테스트 |
| systematic-debugging | 1.버그수정 | 체계적 디버깅 프로세스 |
